# Quick Comments for Leads and Tickets
Status: completed
## 1) What this module is
Quick Comments lets staff add short updates to leads and support tickets so everyone can see the latest status.

## 2) What staff can do here
- Add quick status notes after calls or messages
- Record next actions and follow-up dates
- Share updates with teammates in one place
- Keep a clear history of what was done
- Reduce repeated questions between shifts

## 3) How to use (step-by-step)
1. Open the lead or ticket record.
2. Find the Quick Comment or comment area.
3. Write a short update with key facts.
4. Include what happened, what is next, and by when.

## 4) Important buttons/options
- Add Comment: Starts a new update.
- Save/Post: Publishes your comment.
- Activity History: Shows past comments in time order.
- Mention Teammate: Alerts a specific person if needed.
- Attach File (if available): Adds supporting documents.

## 5) Daily best practices
- Write comments right after each interaction.
- Keep notes short, clear, and factual.
- Include dates and next steps in each important update.
- Read the latest comments before contacting the customer.
- Use polite, professional language in every note.

## 6) Common mistakes to avoid
- Writing comments that are too vague to be useful.
- Forgetting to add the next step or follow-up date.
- Posting duplicate comments for the same action.
- Using informal language that can be misunderstood.
- Adding sensitive details that should not be shared widely.
