# Notes Guide for Office Staff
Status: completed
## 1) What this module is
The Notes module is for internal written updates that help teams remember context and next actions.

## 2) What staff can do here
- Create notes for important observations and updates.
- Link notes to a lead or ticket.
- Edit or delete notes when needed (based on permission).
- Filter notes to find recent or related entries quickly.
- Use notes in activity history so the full team sees the same context.

## 4) Important buttons/options
- `New Note`: add fresh internal context.
- Link/reference options: attach note to the correct record.
- `Edit`: correct or refine content.
- `Delete`: remove wrong or duplicate notes.
- `Filters` / search: quickly find notes by record or keyword.

## 5) Daily best practices
- Keep notes short, factual, and date-aware.
- Always include next step and owner when action is required.
- Link notes to the right lead/ticket so everyone sees them.
- Write notes immediately after calls or discussions.
- Use one note thread per issue instead of scattered entries.

## 6) Common mistakes to avoid
- Writing vague notes like “followed up” without details.
- Forgetting to link notes to the right record.
- Using notes for personal reminders only (instead of team context).
- Deleting notes that are still useful for audit/history.
- Storing critical updates outside the shared system.
