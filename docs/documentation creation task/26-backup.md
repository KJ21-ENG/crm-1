# Backup
Status: completed
## 1) What this module is
This module helps your office keep a safe copy of important data. Backups protect your records if something is deleted by mistake or if there is a system problem.

## 2) What staff can do here
- Create a backup now when needed
- See when the last backup was taken
- Download a backup file for safe storage
- Restore data from an older backup (with approval)
- Check backup history and status

## 4) Important buttons/options
- **Create Backup / Backup Now**: Start a new backup
- **Restore**: Bring back data from a selected backup
- **Download**: Save a backup copy to your device
- **Backup History**: View past backups and their status
- **Refresh**: Update the status list

## 5) Daily best practices
- Check backup status at the start of the day.
- Run a manual backup before major updates or bulk changes.
- Keep backup names and dates clear.
- Follow office approval rules before any restore.
- Report failed backups immediately.

## 6) Common mistakes to avoid
- Running restore without permission
- Not checking whether the backup completed successfully
- Deleting old backups without policy approval
- Assuming auto-backup is always running without verifying
- Forgetting to back up before large data edits
