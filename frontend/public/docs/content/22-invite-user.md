# Invite User
Status: completed
## 1) What this module is
Invite User is used to send secure account invitations to new staff. It helps onboard people with the correct access from day one.

## 2) What staff can do here
- Send an invitation email to a new employee.
- Set the initial role before first login.
- Track whether the invite was accepted.

## 3) How to use (step-by-step)
1. Open **Invite User**.
![Screenshot](/docs/screenshots/22-invite-user/01-open-invite-user.png)
2. Enter the employee's official work email.
![Screenshot](/docs/screenshots/22-invite-user/02-enter-email.png)
3. Select the correct team or department details.
4. Assign the approved starting role.
![Screenshot](/docs/screenshots/22-invite-user/03-assign-role.png)
5. Review the invite message.
6. Select **Send Invite**.
![Screenshot](/docs/screenshots/22-invite-user/04-send-invite.png)
7. Follow up if the invite is not accepted in time.

## 4) Important buttons/options
- **Send Invite**: Sends the invitation email.
- **Role selection**: Sets initial access.
- **Resend Invite**: Sends the invitation again.
- **Invitation status**: Shows pending or accepted state.

## 5) Daily best practices
- Confirm manager approval before inviting.
- Use only verified company email addresses.
- Check pending invites daily and resend when needed.

## 6) Common mistakes to avoid
- Sending invites to personal or incorrect email addresses.
- Assigning the wrong starting role.
- Inviting a user twice instead of checking existing records.
- Forgetting to monitor unaccepted invitations.
