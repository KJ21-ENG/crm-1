# Support Pages Guide for Office Staff
Status: completed
## 1) What this module is
The Support Pages module stores approved support links/pages that staff can quickly send to customers.

## 2) What staff can do here
- Create new support links/pages.
- Edit existing link names or URLs.
- Search and find the right support link quickly.
- Delete outdated or incorrect links (if permitted).
- Use these links during customer conversations, including ticket-related support chats.

## 3) How to use (step-by-step)
1. Open `Support Pages`.
![Screenshot](/docs/screenshots/04-support-pages/01-open-support-pages.png)
2. Click `New` to add a support page name and link.
![Screenshot](/docs/screenshots/04-support-pages/02-new-support-page-form.png)
3. Check that the link opens correctly before saving.
![Screenshot](/docs/screenshots/04-support-pages/03-fill-support-page-details.png)
4. Use search/filter to locate links while helping customers.
![Screenshot](/docs/screenshots/04-support-pages/04-search-filter-support-pages.png)
5. Copy/send the selected support link in chat, email, or support conversation.
6. Edit links when content changes.
![Screenshot](/docs/screenshots/04-support-pages/05-edit-support-page.png)
7. Remove duplicate or expired links if your role allows deletion.
![Screenshot](/docs/screenshots/04-support-pages/06-delete-support-page.png)

## 4) Important buttons/options
- `New Support Page`: add a reusable support link.
- `Search` / `Filter`: find links fast during live calls/chats.
- `Edit`: update page name or URL.
- `Delete`: remove wrong or outdated links.
- Active/availability options (if configured): keep only valid links usable.

## 5) Daily best practices
- Test newly added links before sharing with customers.
- Use clear names so teammates can find pages quickly.
- Keep only current links active.
- Standardize commonly sent links to reduce typing errors.
- Review old links regularly for broken destinations.

## 6) Common mistakes to avoid
- Saving broken or incomplete URLs.
- Creating multiple links for the same page with confusing names.
- Sending outdated support links.
- Deleting useful links without team confirmation.
- Keeping inactive content mixed with live customer links.
