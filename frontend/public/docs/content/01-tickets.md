# Tickets Guide for Office Staff
Status: completed
## 1) What this module is
The Tickets module is where you track customer problems from first report to final closure.

## 2) What staff can do here
- Create tickets, usually from `Call Logs` by opening a call detail popup and clicking `Create Ticket`.
- Open a ticket form with call details already filled (like phone number, caller name if available, and call context).
- Set and update ticket status.
- Set priority: `Low`, `Medium`, `High`, or `Urgent`.
- Assign tickets by your office process (auto assignment, direct assignment, or assignment request).
- Call, email, or message customers from the ticket.
- Add comments, notes, tasks, and attachments.
- Review activity history and close resolved issues.
- Delete tickets only if your role allows it.

## 4) Important buttons/options
- `Create Ticket` (in `Call Logs` call popup): start a new ticket from that call.
- Prefilled fields in ticket form: save time and reduce typing mistakes.
- `New Ticket`: manual ticket creation when needed (if not starting from Call Logs).
- `Filters` / saved views: find open, urgent, or team-specific tickets quickly.
- `Status`: track current stage (for example New, Open, In Progress, Follow-up, Resolved, Closed).
- `Priority`: set urgency for response order.
- `Assign` / `Reassign`: set owner for action.
- `Request Assignment`: ask for a specific assignee when approval is needed.
- `Call`, `Email`, `WhatsApp Support`: contact customer from the ticket workflow.
- `Add Comment`, `Add Note`, `Add Task`, `Attach File`: keep complete context in one place.
- `Delete`: available only to authorized roles.

## 5) Daily best practices
- Start most ticket creation from `Call Logs` so call details are prefilled automatically.
- Start the day by checking `New`, `Open`, and `Urgent` tickets first.
- Keep status current so team handoffs are clear.
- Record every customer interaction in the activity timeline.
- Attach proof (screenshots, documents, recordings) when relevant.
- Reassign early if a ticket is blocked.
- Close tickets only after confirming the issue is resolved.

## 6) Common mistakes to avoid
- Skipping the `Call Logs` popup flow and missing call details in the ticket.
- Creating duplicate tickets for the same issue.
- Leaving tickets without an assignee.
- Forgetting to update status after calls or emails.
- Closing a ticket without resolution notes.
- Using low priority for urgent customer issues.
- Keeping key details in personal chat instead of inside the ticket.
